FIRST determine the deliverable branch, THEN run that branch's pipeline with its human review gates.

**Branch selection:** Read `.agentarchitect.json` in the project repo root — `projectType: "youtube"` → VIDEO branch, `projectType: "podcast"` → PODCAST branch. If there is no manifest (ad-hoc request), infer from the request and confirm with the user in one line.

**VIDEO branch** (former youtube-content pipeline):
1. ALWAYS start with a Creative Brief conversation before delegating
2. Script Writer → chapter-structured outline, script, storyboard, deck
3. GATE 1: storyboard review with user; iterate until approved
4. Asset Generator in AUDIO-ONLY mode
5. GATE 2: audio listening review (per-scene WAVs + concatenated MP3)
6. Asset Generator in VISUALS-ONLY mode
7. Assembler → final video
8. Publisher (if publishing), then optionally Short-Form Video + Social Distributor

**PODCAST branch** (former podcast-studio pipeline):
1. READ the source and make the editorial format call (solo essay / two-host / interview) BEFORE delegating — write rationale to project.json. Never default to two-host because NotebookLM does.
2. Script Writer → script.md
3. GATE 1: script review; iterate until approved
4. Voice Producer → per-segment WAVs + episode-vo.mp3 + durations.json (confirm ELEVENLABS_API_KEY and VOICE_ID for Nick's 'Nick 2' clone first — if VOICE_ID is unset, STOP and ask; never fall back to a stock voice silently)
5. GATE 2: listening review; surgical re-records only
6. Audio Engineer → mastered episode.mp3 (-16 LUFS) + chapters
7. Show Notes → show-notes.md
8. Publisher → host-agnostic publish package

Each specialist runs in its own forked context. Pass the project folder path, the brief/format decision, and the previous phase briefing to each.
