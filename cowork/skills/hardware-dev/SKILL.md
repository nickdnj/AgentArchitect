# Hardware Development Team - Orchestrator

Hardware product development team covering schematic/PCB design, mechanical engineering, firmware, supply chain, and manufacturing readiness

## Orchestration Strategy

Route hardware tasks to the appropriate specialist. For new product initiatives, start with systems-engineer for ConOps, requirements decomposition, system architecture, and interface specs. Systems-engineer then allocates requirements to specialists. For detailed design, fan out to pcb-designer and mcad-engineer in parallel, with firmware-engineer for pin mapping. Supply chain follows design. DFM is the final gate. Systems-engineer coordinates cross-team integration with software-project team for HW/SW products. For focused tasks on existing designs (e.g., 'update the schematic'), route directly to the specialist.

## Workflow Stages

**Systems Engineering**: Systems Engineering Manager
  - Outputs: ConOps, System requirements, System architecture, Interface specifications, V&V plan, Traceability matrix
**Requirements Allocation**: Systems Engineering Manager
  - Inputs: System requirements, System architecture
  - Outputs: Electronics requirements, Mechanical requirements, Firmware requirements, Software requirements, Pin mapping, Interface specification
**Design** (parallel): PCB Designer, MCAD Engineer
  - Inputs: Hardware requirements, Pin mapping
  - Outputs: Schematic, PCB layout, Gerbers, Enclosure STEP, Board STEP
**Sourcing**: Supply Chain Manager
  - Inputs: Electronic BOM, Mechanical BOM
  - Outputs: Costed BOM, CM quotes, Risk assessment
**Implementation**: Firmware Engineer
  - Inputs: Final schematic, Pin mapping
  - Outputs: Firmware binary, Test procedures, Flash instructions
**Validation**: DFM & Test Engineer
  - Inputs: Gerbers, Enclosure design, Firmware test mode, CM capabilities
  - Outputs: DFM report, Test strategy, Test fixture spec, Compliance checklist
