**Key MVP decisions:**
- EMQX Cloud free tier as MQTT v5 broker (not AWS IoT Core)
- Firebase for auth, Firestore, Cloud Functions, FCM (not AWS)
- No HTTP/WebSocket server on device
- No OTA firmware updates in v1.0
- PlatformIO with ESP-IDF framework (not Arduino)

## 2. Firmware Task Architecture

```
CORE 1 (dedicated sensor)          CORE 0 (application)
┌──────────────────────┐           ┌────────────────────────────────────┐
│ SENSOR READ TASK     │           │ SAFETY MONITOR TASK                │
│ Priority: 20         │           │ Priority: 24 (HIGHEST)             │
│ Stack: 4096 bytes    │           │ Stack: 4096 bytes                  │
│ Period: 500ms (2Hz)  │           │ Blocks on: xQueueReceive()         │
│                      │           │                                    │
│ 1. I2C read MLX90640 │           │ 1. Receive stove_event_t           │
│    (both subpages)   │           │ 2. Update rolling ambient from     │
│ 2. Sanity check frame│           │    coldest 10% of pixels           │
│    (reject all-zero, │           │ 3. Evaluate ON/OFF with hysteresis │
│     all-0xFFFF,      │           │    ON: max > ambient + 50°C        │
│     flat frames)     │           │    OFF: max < ambient + 35°C       │
│ 3. Extract max_temp  │           │ 4. Debounce (3 frames ON, 6 OFF)   │
│ 4. Extract bottom    │           │ 5. Run unattended timer            │
│    10th percentile   │           │ 6. Fire buzzer on timeout          │
│ 5. Build stove_      │           │ 7. Send alert_msg_t to WiFi task   │
│    event_t           │           │ 8. Handle test_buzzer commands     │
│ 6. xQueueSend()     │           │                                    │
└──────────┬───────────┘           └──────────┬─────────────────────────┘
           │ event_queue (depth:4)             │ alert_queue (depth:8)
           └──────────▶ SAFETY MONITOR         └──────────▶ WiFi/MQTT TASK
                                                            │
                                               ┌────────────┘
                                               ▼
                                  ┌────────────────────────────────────┐
                                  │ WIFI / MQTT TASK                   │
                                  │ Priority: 10                       │
                                  │ Stack: 8192 bytes (TLS needs 6KB)  │
                                  │                                    │
                                  │ 1. WiFi auto-reconnect (exp back)  │
                                  │ 2. MQTT v5 connection + keepalive  │
                                  │ 3. Publish alerts (QoS 1)          │
                                  │ 4. Publish heartbeat (every 60s)   │
                                  │ 5. Subscribe to command topic      │
                                  │ 6. Buffer to NVS on disconnect     │
                                  │ 7. Flush NVS buffer on reconnect   │
                                  │ 8. Publish reconnect summary       │
                                  │ 9. I2C health telemetry            │
                                  └────────────────────────────────────┘

DATA STRUCTURES:
────────────────
stove_event_t  {max_temp, ambient_temp, timestamp_ms}          12 bytes
alert_msg_t    {alert_type, max_temp, duration_sec, ts, id}    ~40 bytes
stove_state_t  {is_on, on_since, alert_active, debounce_count} ~16 bytes

SYNCHRONIZATION:
────────────────
event_queue    Sensor → Safety Monitor     (depth 4, 100ms timeout)
alert_queue    Safety Monitor → WiFi Task  (depth 8, portMAX_DELAY)
wifi_events    xEventGroupBits             (WIFI_CONNECTED | MQTT_CONNECTED)
buzzer_mutex   xSemaphoreMutex             (guards buzzer GPIO)
nvs_mutex      xSemaphoreMutex             (guards NVS read/write)
```

## 3. State Machine — Stove Detection

```
                         Power On / Reboot
                              │
                              ▼
                    ┌───────────────────┐
                    │   INITIALIZING    │
                    │ Read NVS config   │
                    │ Capture 3 frames  │
                    │ Set ambient base  │
                    └────────┬──────────┘
                             │ 3 stable frames
                             ▼
              ┌──────────────────────────────────┐
              │           STOVE_OFF              │◀─────────────────────┐
              │ max < ambient + 50°C             │                      │
              │ (rolling ambient from coldest    │                      │
              │  10% of pixels per frame)        │                      │
              └──────────────┬───────────────────┘                      │
                             │ max > ambient + 50°C                     │
                             │ for 3 consecutive frames (1.5s)          │
                             ▼                                          │
              ┌──────────────────────────────────┐                      │
              │           STOVE_ON               │                      │
              │ • Record on_since = now          │                      │
              │ • Send STOVE_ON alert            │                      │
              │ • Start duration counter         │                      │
              │ • LED → green                    │                      │
              └──────┬──────────────┬────────────┘                      │
     duration >=     │              │ max < ambient + 35°C              │
     30 min          │              │ for 6 frames (3s)                 │
                     ▼              ▼                                    │
      ┌──────────────────┐  ┌──────────────┐                           │
      │ UNATTENDED_WARN  │  │ STOVE_COOLING│                           │
      │ • Send WARNING   │  │ • Wait 60s   │                           │
      │ • LED → amber    │  │   to confirm │                           │
      └──────┬───────────┘  └──────┬───────┘                           │
             │ +30 min unack'd     │ still below threshold              │
             ▼                     │ after 60s                          │
      ┌──────────────────┐         └────────────────────────────────────┘
      │ UNATTENDED_CRIT  │                Send STOVE_OFF, reset timer
      │ • ACTIVATE BUZZER│
      │ • Send CRITICAL  │
      │ • LED → red      │
      └──────────────────┘
        Buzzer silenced by: MQTT ack, button hold, or stove turns off

THRESHOLDS (configurable via MQTT or BLE):
  ON:  ambient + 50°C (with rolling ambient)
  OFF: ambient + 35°C (asymmetric hysteresis)
  DEBOUNCE_ON:  3 frames (1.5s at 2Hz)
  DEBOUNCE_OFF: 6 frames (3s)
  WARNING_TIMEOUT: 30 min (default)
  CRITICAL_TIMEOUT: 60 min (default)
  COOLING_CONFIRM: 60s
```

## 4. BLE Provisioning Sequence

```
  FLUTTER APP                        ESP32-S3                         ROUTER
  ──────────                         ────────                         ──────
      │ Tap "Add Device"                 │                               │
      │── BLE Scan (filter StoveIQ-*) ──▶│ Advertising                   │
      │◀── Found: StoveIQ-A1B2 ─────────│                               │
      │── GATT Connect ────────────────▶│                               │
      │── ECDH Key Exchange ───────────▶│                               │
      │◀── Device pubkey + device_id ───│                               │
      │ Derive shared secret             │ Derive shared secret          │
      │ User selects SSID, enters pass   │                               │
      │── Encrypted creds (AES-128) ───▶│                               │
      │                                  │── WiFi Connect ──────────────▶│
      │                                  │◀── DHCP IP ──────────────────│
      │                                  │── MQTT Connect ──▶ Broker     │
      │◀── BLE: OK + IP + MQTT status ──│                               │
      │ Register device with Firebase    │                               │
      │── BLE: Provisioning Complete ──▶│ Stop BLE, free memory          │
      │ Navigate to status screen        │ Enter normal operation         │

  ERROR CODES:
    0x02 WIFI_AUTH_FAIL    → "Incorrect password"
    0x03 WIFI_SSID_NOT_FOUND → "Network not found"
    0x04 WIFI_DHCP_TIMEOUT → "Connected but no IP"
    0x05 MQTT_FAIL         → "WiFi OK, cloud unreachable (will retry)"
```

## 5. Happy Path Latency Analysis

```
Burner ON → Sensor read (500ms) → State eval (1ms) → MQTT publish (50ms)
→ Cloud Function (50ms) → FCM delivery (500ms)
= ~1.1s typical, ~39s worst case

Worst case dominated by: FCM on Android Doze mode (up to 10 min)
→ Local buzzer is the TRUE safety backstop, not push notification
```

## 6. Failure Mode Analysis

| # | Failure | Consequence | Handling | User Sees | Critical? |
|---|---------|-------------|----------|-----------|-----------|
| 1 | I2C bus hang | State frozen | HW watchdog reboot (10s) | Device restarts | Fixed |
| 2 | Flat thermal frame | False "stove off" | Reject frame, error counter, fault after 10 | Sensor Error | Fixed |
| 3 | WiFi dropout | No push notifications | Buzzer still works. Buffer to NVS. Reconnect summary on restore. | App: "Offline" | Fixed |
| 4 | MQTT disconnect | Alerts not reaching cloud | Buffer + retry + NVS overflow to disk | Buzzer works, gap in history | Fixed |
| 5 | Cloud Function crash | No push sent | Auto-retry 3x. Dead letter queue. | **SILENT** — buzzer is backup | Accepted |
| 6 | FCM not delivered | Phone in Doze/airplane | No retry possible | **SILENT** — buzzer is backup | Accepted |
| 7 | Power loss | Total monitoring loss | Heartbeat at 60s. Cloud detects offline in 3 min. | "Device Offline" push after 3 min | Accepted for MVP |
| 8 | Wrong threshold | False positives/negatives | Adaptive rolling ambient + hysteresis | Configurable via settings | Fixed |

**Critical gaps (none after fixes):** All failure modes either have handling or an accepted fallback (buzzer).

## 8. Technology Stack — Final MVP

| Layer | Choice | Rationale |
|-------|--------|-----------|
| MCU | ESP32-S3-WROOM-1-N8R8 | Dual core, WiFi, BLE, 8MB PSRAM |
| Sensor | MLX90640 (110° FOV BAB) | Order both FOV variants, design for 110° |
| Framework | ESP-IDF v5.3.x via PlatformIO | Full FreeRTOS control, BLE provisioning |
| RTOS | FreeRTOS (3 tasks) | Safety Monitor must preempt WiFi |
| Protocol | MQTT v5 / TLS | Persistent connection, push alerts, QoS 1 |
| Broker | EMQX Cloud free tier | No X.509 cert infra for MVP |
| Cloud | Firebase (Auth, Firestore, Functions, FCM) | One SDK, free tier covers MVP |
| App | Flutter (FlutterFire + flutter_blue_plus) | Best BLE library, path to heat map |
| Build | PlatformIO | Faster than raw ESP-IDF CMake |

## 10. Parallelization Strategy
