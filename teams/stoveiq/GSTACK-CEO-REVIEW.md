## Critical MVP Scope Reduction

The CEO review identified that the original "Safety MVP" was still too broad. The true MVP answers one question: **"Is Mom's stove on?"**

**Stripped from MVP:**
- Heat map rendering → v2
- Per-burner detection → v2
- Burner state machine → v2
- Perspective correction / homography → v2
- Calibration wizard → v2
- Boil detection → v2
- Cooking session history → v2

**What remains:**
- Whole-frame max temperature → stove ON/OFF binary
- Duration timer → unattended alert
- Push notification → caregiver's phone
- Local buzzer → on-device safety net

## Key Technical Decisions (Resolve NOW)

1. **MLX90640 FOV variant:** Order BOTH 110° (BAB) and 55° (BAA). Test both. Design for 110° — under-cabinet at 24" needs wider FOV.
2. **ESP-IDF version:** Pin to v5.3.x stable. Do NOT use v5.4 beta — MLX90640 driver has known I2C issues.
3. **I2C speed:** Start at 400kHz, not 1MHz. Many breakout boards lack adequate pull-ups for 1MHz.
4. **Ambient baseline:** Cold-boot capture stored in NVS, updated only when all burners OFF >10 min.
5. **Unattended detection:** Simple duration timer, NOT thermal-change analysis. Simmering is not "unattended."
6. **Calibration:** Skip entirely for Safety MVP. Whole-frame max temperature doesn't need burner positions.
7. **Test rig:** Budget $15 for a desktop hotplate + smart plug for automated sensor testing.
